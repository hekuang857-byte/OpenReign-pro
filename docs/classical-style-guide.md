# 古风使用指南

## 概述

OpenReign Pro 采用**古代三省六部制**架构，所有术语、消息、界面均遵循古风设计。

## 核心原则

```
现代术语 → 古代官职
技术概念 → 古代用语
英文缩写 → 中文全称
直白描述 → 文言奏折
```

## 代码中的古风使用

### 上报进度

```javascript
// 使用古风格式化
await kanban.progressClassical(
  task.id,
  'zhongshu',           // 部门ID
  '辨析',               // 古风动作
  '拆解核心要点',        // 详情
  '分析🔄|起草|审议'     // 计划
);

// 输出：启奏陛下，中书省正在辨析，拆解核心要点
```

### 记录流转

```javascript
// 自动古风化
await kanban.flow(
  task.id,
  'zhongshu',           // 自动转为"中书省"
  'menxia',             // 自动转为"门下省"
  '📋 方案提交审议'      // 自动古风化
);
```

### 格式化消息

```javascript
const formatter = new ClassicalFormatter();

// 转换现代术语
const classical = formatter.modernToClassical(
  '任务执行完成，代码已部署到服务器'
);
// 输出：奏折施行告竣，文书已安置到机房

// 格式化进度
const progress = formatter.formatProgress(
  'bingbu',
  '施行',
  '攻城略地'
);
// 输出：启奏陛下，兵部正在施行，攻城略地
```

## 注意事项

1. **保持一致性** - 同一文档/消息中统一使用古风
2. **适度古风** - 关键术语古风化，保持可读性
3. **避免过度** - 不要强行翻译所有词汇
4. **保留专业** - 技术概念可以适当保留现代用法
