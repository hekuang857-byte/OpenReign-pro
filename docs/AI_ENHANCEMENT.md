# AI 能力融入三省六部方案

## 核心原则
- **不新增部门**：保持现有三省六部架构
- **技能化**：我的能力作为六部的技能插件
- **看得懂**：每个部门增加什么能力一目了然

---

## 一、太子（任务分发）+ AI 意图识别

**现状**：太子根据关键词判断任务类型

**增强**：AI 自然语言理解

```
用户输入 → AI理解意图 → 太子判断复杂度 → 分流

示例：
用户："帮我看看这个bug"
AI理解：代码审查任务 → 太子判断复杂度4 → 分流到：中书省规划 → 兵部执行
```

**实现**：太子部门增加 `ai_intent` 技能

---

## 二、中书省（决策规划）+ AI 任务拆解

**现状**：人工配置任务流程

**增强**：AI 自动规划执行步骤

```
任务：添加用户登录功能
AI规划：
  1. 兵部：创建 auth.ts 核心代码
  2. 户部：设计用户表结构
  3. 礼部：写 API 文档
  4. 刑部：安全审查
  5. 工部：部署配置
```

**实现**：中书省增加 `ai_planning` 技能

---

## 三、门下省（审核监督）+ AI 代码审查

**现状**：人工审核

**增强**：AI 自动审查代码

```
兵部提交代码 → 门下省AI审查 → 发现问题 → 退回修改

审查内容：
- 代码规范
- 潜在bug
- 安全漏洞
- 性能问题
```

**实现**：门下省增加 `ai_review` 技能

---

## 四、尚书省（执行调度）+ AI 进度预测

**现状**：固定调度

**增强**：AI 预测任务耗时和资源需求

```
任务队列：
- 任务A：预计15分钟，需兵部
- 任务B：预计30分钟，需户部+兵部
- AI调度：先执行A，B并行准备
```

**实现**：尚书省增加 `ai_schedule` 技能

---

## 五、六部执行 + AI 代码生成

### 兵部（代码执行）
**增强技能**：
- `ai_code_gen`：根据需求生成代码
- `ai_refactor`：自动重构代码
- `ai_test_gen`：生成单元测试

### 礼部（文档沟通）
**增强技能**：
- `ai_doc_write`：自动生成文档
- `ai_comment`：自动写代码注释

### 户部（数据记忆）
**增强技能**：
- `ai_data_model`：自动设计数据模型
- `ai_query_opt`：优化查询语句

### 工部（部署运维）
**增强技能**：
- `ai_deploy_script`：生成部署脚本
- `ai_monitor`：智能监控告警

### 刑部（安全审计）
**增强技能**：
- `ai_security_scan`：安全扫描
- `ai_compliance_check`：合规检查

---

## 六、军机处（定时任务）+ AI 自动化

**现状**：固定定时任务

**增强**：AI 智能触发

```
原：每5分钟扫描兵器库
新：AI检测代码变更 → 自动触发扫描

原：每小时Token统计
新：AI预测高峰期 → 动态调整统计频率
```

**实现**：军机处增加 `ai_trigger` 技能

---

## 七、外史院（外部渠道）+ AI 智能回复

**现状**：消息转发

**增强**：AI 预处理回复

```
用户@飞书机器人："系统挂了"
AI预处理：
  1. 查询系统状态
  2. 生成回复："检测到xxx问题，正在处理"
  3. 转发太子创建任务
```

**实现**：外史院增加 `ai_response` 技能

---

## 实现方式

### 1. 配置层面（推荐）

在 `openreign.json` 中给部门增加 AI 技能：

```json
{
  "agents": {
    "bingbu": {
      "skills": ["terminal", "github", "ai_code_gen", "ai_refactor"],
      "ai_features": {
        "auto_code": true,
        "auto_test": true
      }
    },
    "menxia": {
      "skills": ["audit", "ai_review"],
      "ai_features": {
        "auto_review": true
      }
    }
  }
}
```

### 2. 界面展示

在部门卡片上增加 AI 标识：

```
┌─────────────┐
│   兵部 🤖   │  ← AI增强标识
│  代码执行   │
│             │
│ AI技能：    │
│ ·代码生成   │
│ ·自动重构   │
│ ·测试生成   │
└─────────────┘
```

### 3. 工作流程示例

**用户**："修复登录bug"

**流程**：
1. 太子 + AI意图识别 → 理解是bug修复任务
2. 中书省 + AI规划 → 拆解步骤
3. 兵部 + AI代码生成 → 自动修复代码
4. 门下省 + AI审查 → 自动审核
5. 尚书省 → 调度完成
