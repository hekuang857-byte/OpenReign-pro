## 📊 案例3: 数据分析任务 (封驳重试)

### 场景
用户说："分析这份销售数据，找出增长原因" (数据文件过大)

### 预期
门下省第一次审核驳回(数据过大)，中书省调整后重新提交，第二次通过。

### 流程图

```
第一次提交:
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  皇上   │───▶│  太子   │───▶│ 中书省  │───▶│ 门下省  │
│ (用户)  │    │ (分拣)  │    │ (规划)  │    │ (审核)  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │
     │ 下旨:分析数据 │ 识别:数据分析 │ 规划:完整分析 │ 审核:❌驳回
     │              │              │              │ 原因:数据过大
     ▼              ▼              ▼              ▼
   0ms            100ms          600ms          900ms
                                                 │
                                                 ▼
                                           ┌─────────┐
                                           │  封驳   │
                                           │ (驳回)  │
                                           └─────────┘
                                                 │
                                                 ▼
                                           返回中书省

第二次提交(调整后):
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 中书省  │───▶│ 门下省  │───▶│ 尚书省  │───▶│   户部  │───▶│  完成   │
│ (调整)  │    │ (再审)  │    │ (派发)  │    │ (分析)  │    │ (回奏)  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     │ 调整:分批处理 │ 审核:✅通过  │ 派发:户部    │ 分析完成     │ 汇总结果
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
   1000ms         1300ms         1500ms         5000ms         5500ms

总耗时: 8.2秒 (含封驳)
成本: $0.078
审议轮次: 2轮
```

## 📊 案例4: 定时任务 (钦天监执行)

### 场景
用户设置："每天早上9点，自动检查服务器状态"

### 预期
任务创建后进入钦天监，定时触发执行。

### 流程图

```
创建阶段:
┌─────────┐    ┌─────────┐    ┌─────────┐
│  皇上   │───▶│  太子   │───▶│ 钦天监  │
│ (用户)  │    │ (分拣)  │    │ (定时)  │
└─────────┘    └─────────┘    └─────────┘
     │              │              │
     │ 设置定时任务  │ 识别:定时任务 │ 注册:每天9点
     │              │              │
     ▼              ▼              ▼
   0ms            100ms          200ms
                                    │
                                    ▼
                              ┌─────────┐
                              │  等待   │
                              │ (挂起)  │
                              └─────────┘
                                    │
                   每天9:00触发      ▼
                                    │
                              ┌─────────┐
                              │  兵部   │
                              │ (执行)  │
                              └─────────┘
                                    │
                                    ▼
                              ┌─────────┐
                              │  完成   │
                              │ (回奏)  │
                              └─────────┘

首次触发: 明天9:00
周期: 每天
成本/次: $0.012
```
