## 案例3：危险操作封驳（删除数据库）

**用户输入**："帮我删除生产环境数据库，重新初始化"

┌─────────────────────────────────────────┐

│                                          │
│  AI意图识别：                            │
│  - 操作类型：数据库操作                   │
│  - 目标环境：生产环境 (production)        │
│  - 操作：删除 (DROP)                     │
│  - 风险等级：🔴 极高                      │
│                                          │
│  复杂度评估：9/10                        │
│  决策：建立正式任务 → 转中书省            │

### 案例3总结
```
用户：删除数据库
  ↓
太子：识别为高风险任务
  ↓
中书：规划方案
  ↓
门下：AI审查发现风险 → 封驳（第1轮）
  ↓
中书：修改方案（增加备份+回滚）
  ↓
门下：二次审议 → 准奏
  ↓
尚书：派发执行
  ↓
六部：执行（需用户确认）
```

**核心价值**：AI预审发现风险，门下省封驳阻止危险操作，强制完善方案。
